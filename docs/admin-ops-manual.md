# VNC PLATFORM — ADMIN OPERATIONS MANUAL
Final Master Hard-Lock Version: v6.7.0.4

---

## 1. Role Definition

An **Admin** is an operational role responsible for:
- Reviewing user activity
- Handling support escalations
- Investigating flagged incidents
- Producing audit and compliance outputs

Admins are **not system owners** and have **no override authority**.

---

## 2. Core Admin Responsibilities

Admins may perform the following actions:
- Review user profiles and KYC status
- View wallet and ledger histories (read-only)
- Investigate fraud or anomaly alerts
- Escalate incidents for freeze or owner review
- Respond to support tickets

---

## 3. Explicit Prohibitions

Admins may NOT:
- Unfreeze users or wallets
- Modify ledger entries
- Change balances
- Bypass zero-trust checks
- Disable security mechanisms
- Activate or deactivate the kill-switch

Any attempt is logged and auditable.

---

## 4. Incident Handling Workflow

### Step 1: Detection
- Alert generated by AI, risk matrix, or user report

### Step 2: Review
- Admin reviews activity timeline
- Admin verifies risk indicators

### Step 3: Escalation
- Admin escalates incident
- System executes freeze automatically

### Step 4: Documentation
- Forensic snapshot generated
- Admin documents findings

---

## 5. KYC Review Process

Admins may:
- Review submitted KYC data
- Approve or reject KYC applications

Rules:
- Decisions are final unless owner intervention occurs
- All decisions are logged
- KYC state gates restricted features automatically

---

## 6. Transaction Review

Admins may:
- View transaction histories
- Flag suspicious patterns

Admins may NOT:
- Reverse transactions
- Edit transaction data

---

## 7. Support Ticket Handling

Admins may:
- View support tickets
- Respond with guidance
- Escalate issues to security or owner

Admins may NOT:
- Access unrelated user data
- Modify system state via support actions

---

## 8. Audit & Reporting

Admins can generate:
- Audit summaries
- Compliance reports
- Incident timelines

Reports are read-only and reproducible.

---

## 9. Accountability & Logging

- All admin actions are logged
- Logs are immutable
- Logs are subject to audit review

Misuse of admin privileges results in access revocation.

---

## 10. Emergency Scenarios

Admins must:
- Escalate immediately
- Avoid independent action
- Follow freeze-first policy

Admins cannot resolve emergencies unilaterally.

---

## 11. Training & Access

- Admin access is granted explicitly
- Least-privilege principles apply
- Periodic review of admin access is required

---

## 12. Seal Statement

Any modification to admin permissions or capabilities
invalidates this manual and breaks the hard-lock.
